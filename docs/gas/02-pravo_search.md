# pravo_search — Поиск судебных дел

POST `https://api.newdb.net/v2`

Метод ищет судебные дела в ГАС Правосудие и связанных источниках судов общей юрисдикции.
В публичном описании для поиска используется один основной параметр: `query`.

В ответе возвращается список найденных дел с карточкой дела, списком участников и списком актов. Для дальнейшей детализации конкретного дела используйте [pravo_cases_details](01-pravo_cases_details.md).

## Когда использовать

Используйте метод, когда нужно найти судебные дела по общему поисковому тексту: ФИО, названию организации, номеру дела, фрагменту карточки дела или другому текстовому запросу.

## Типовые кейсы

- Поиск дел по ФИО физлица
- Поиск дел по организации
- Поиск дел по номеру дела
- Поиск дел по общему текстовому запросу

## Входная схема (request)

```json
{
  "params": {
    "query": "string",
    "country": "ru",
    "method": "pravo_search"
  },
  "requestId": "optional-string"
}
```

## Как работает поиск

По коду метод принимает `query` и использует его как общий текстовый поисковый параметр.

- Поиск возвращает список дел, отсортированный по `case_id` по убыванию
- В ответ попадают карточка дела, `parties` и `acts`
- Метод поддерживает пагинацию через `page` и `page_size` / `limit`
- Для дальнейшей детализации конкретного дела используйте `pravo_cases_details`

## Описание параметра

| Параметр | Обязательность | Назначение | Какие значения принимает |
|------|------|------|------|
| `query` | Да | Основной поисковый текст для поиска судебных дел | Любой текст: ФИО, организация, номер дела, фрагмент карточки |
| `page` | Нет | Номер страницы результатов | Целое число от `1`. Алиасы: `page_number`, `pageNumber`. По умолчанию `1` |
| `page_size` | Нет | Размер страницы | Целое число от `1` до `20`. Алиасы: `pageSize`, `limit`. По умолчанию `20` |

## Пагинация

Метод `pravo_search` реализован в `ods_case_search.py` и использует постраничную выдачу через `page` и `page_size`.

- `page` начинается с `1`.
- Максимальный `page_size` — `20`. Если передать больше `20`, метод ограничит значение до `20`.
- Вместо `page_size` можно передать `limit`; метод использует его как размер страницы.
- Внутреннее смещение считается как `(page - 1) * page_size`.
- На каждую следующую страницу отправляйте новый запрос с новым `requestId`.

Пример последовательности:

```text
1-я страница: page=1, page_size=20
2-я страница: page=2, page_size=20
3-я страница: page=3, page_size=20
```

Пример запроса второй страницы:

```json
{
  "params": {
    "query": "Антосенко Александр Иванович",
    "country": "ru",
    "method": "pravo_search",
    "page": 2,
    "page_size": 20
  },
  "requestId": "b2b13d26-1c10-4e35-a60e-7f38c73064bc"
}
```

В ответе блок `meta` содержит поля пагинации:

- `page` — номер запрошенной страницы.
- `page_size` — фактический размер страницы.
- `offset` — рассчитанное смещение.
- `total_count` — общее количество найденных дел по фильтру.
- `returned_count` — сколько дел вернул текущий ответ.
- `has_more` — есть ли следующая страница.
- `pages_count` — всего страниц при текущем `page_size`.

Останавливайте обход, когда `meta.has_more = false` или текущий ответ вернул пустой `data`.
